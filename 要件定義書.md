# Frontend Reference App 要件定義書

## 1. 概要

本プロジェクトはチケット管理システムを題材にしたフロントエンド設計リファレンス実装である。

実運用を目的としたシステムではなく、以下の設計・開発プラクティスをチーム内へ共有するためのサンプルプロジェクトとして作成する。

* Feature Directory Architecture
* Atomic Design
* Next.js App Router
* Material UI
* Material React Table
* Plotly
* Storybook
* Interaction Test
* Vitest
* Custom Hooks
* API Layer分離

---

## 2. プロジェクト目的

本プロジェクトでは以下を実現する。

### 目的

* コンポーネント責務の分離例を示す
* Feature単位でのディレクトリ設計例を示す
* APIアクセスの抽象化例を示す
* Storybook運用例を示す
* テスト戦略のサンプルを示す

### 対象読者

* フロントエンドエンジニア
* 新規参画メンバー
* 設計レビュー担当者

---

## 3. システム概要

簡易的なチケット管理システムを構築する。

ユーザーはチケットを作成・閲覧・更新できる。

またチケット情報を集計し、分析画面で可視化できる。

  description: string;

  updatedAt: string;

## 14. 設計原則

* Feature First
* PresentationとData Fetchingの分離
* Container/Presentationalの考え方を採用
* Hooksで状態管理を集約
* Componentsは可能な限りPureに保つ
* 型をAPIレスポンスと同期させる
